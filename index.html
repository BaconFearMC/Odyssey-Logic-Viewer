Kingdom folder, no index file needed

There's no index.json to maintain anymore. The kingdom list, filenames, and correct story order are all baked into the KINGDOM_MANIFEST list at the top of the script below — e.g. { file: "cap_kingdom", name: "Cap Kingdom" }. On page load, it simply requests logic/cap_kingdom.json, logic/cascade_kingdom.json, logic/sand_kingdom.json, etc. straight down that list; any kingdom whose file isn't there yet is silently skipped (no error), so you can fill in the folder gradually. To add a kingdom that isn't in the list, add one entry to KINGDOM_MANIFEST (and drop the matching file in /logic/) — its tab order is whatever position you put it in the list.

Kingdom tab order

Tabs always render in KINGDOM_MANIFEST order (the games's own Cap → Cascade → Sand → Lake → Wooded → Cloud → Lost → Metro → Snow → Seaside → Luncheon → Ruined → Bowser's → Moon → Mushroom → Dark Side → Darker Side sequence), regardless of what order the files finish loading in or what order you pick them in the file picker. Want a different order? Just reorder the entries in KINGDOM_MANIFEST.

Moon order within a kingdom

Moons are sorted automatically using each row's shine_info_hint_idx — the same index the in-game Moon List uses — so "Our First Power Moon", "Multi Moon Atop the Falls", etc. come out in the real game order, not whatever order they happened to sit in the export file. Any moon missing that field keeps its original position relative to its neighbors.

Icon art

Put files in /icon/ named after the Capture or Ability's own display name, in kebab-case (e.g. Chain Chomp → chain-chomp.png, T-Rex → t-rex.png, Ground Pound → ground-pound.png). Until a matching file exists, a colored placeholder chip with the item's initials shows instead (no emoji, ever) — nothing breaks, and dropping in real art later needs zero code changes.

Checkpoint & Subarea icon reference

A moon card shows exactly one of these in the bottom-right corner, at the same size as a Capture icon. If the moon is inside a subarea (e.g. Mysterious Clouds, 8-Bit Chasm Lifts), it shows the Subarea picture and hovering/focusing reveals that subarea's name; otherwise it shows the regular Checkpoint icon, and hovering/focusing reveals which Checkpoint the moon is tracked from. Neither name is shown persistently anymore.

Putting this on GitHub

Suggested repo layout:
index.html — this file
icon/*.png — your Capture/Ability icon art
logic/cap_kingdom.json, logic/cascade_kingdom.json, ... — your raw export files, unmodified, named to match KINGDOM_MANIFEST
When this page is opened over http(s):// (which GitHub Pages does, but a double-clicked local file does not), it automatically fetches each file in /logic/ — so once it's pushed and Pages is turned on (repo Settings → Pages → Deploy from branch), all 17 kingdoms just show up with no manual upload step and no index file to keep in sync.
