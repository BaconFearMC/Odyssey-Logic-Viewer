How this stays rinse-and-repeat for all 17 kingdoms

Every export file already contains a logicTokens table (token → display name → category) and a moonAccess table (requirement_expr, from_node_id, shine_info_is_moon_rock, etc). This page reads both directly — it builds the icon manifest from logicTokens and turns each moon's requirement_expr into OR/AND chips by splitting on | and & (respecting parentheses), the same way Timer Challenge 2 becomes 7 separate option-chips. Drop a new kingdom file in /logic/ and its moons appear as a new tab — nothing in this file needs editing.

Kingdom folder, no index file needed

There's no index.json to maintain anymore. The kingdom list, filenames, and correct story order are all baked into the KINGDOM_MANIFEST list at the top of the script below — e.g. { file: "cap", name: "Cap Kingdom" }. On page load, it simply requests logic/cap.json, logic/cascade.json, logic/sand.json, etc. straight down that list; any kingdom whose file isn't there yet is silently skipped (no error), so you can fill in the folder gradually. To add a kingdom that isn't in the list, add one entry to KINGDOM_MANIFEST (and drop the matching file in /logic/) — its tab order is whatever position you put it in the list.

Kingdom tab order

Tabs always render in KINGDOM_MANIFEST order (the games's own Cap → Cascade → Sand → Lake → Wooded → Cloud → Lost → Metro → Snow → Seaside → Luncheon → Ruined → Bowser's → Moon → Mushroom → Dark Side → Darker Side sequence), regardless of what order the files finish loading in or what order you pick them in the file picker. Want a different order? Just reorder the entries in KINGDOM_MANIFEST.

Moon order within a kingdom

Moons are sorted automatically using each row's shine_info_hint_idx — the same index the in-game Moon List uses — so "Our First Power Moon", "Multi Moon Atop the Falls", etc. come out in the real game order, not whatever order they happened to sit in the export file. Any moon missing that field keeps its original position relative to its neighbors.

Icon art

Put files in /Icon/ named after the Capture or Ability's own display name, in kebab-case (e.g. Chain Chomp → chain-chomp.png, T-Rex → t-rex.png, Ground Pound → ground-pound.png). Until a matching file exists, a colored placeholder chip with an emoji shows instead — nothing breaks, and dropping in real art later needs zero code changes.

Putting this on GitHub

Suggested repo layout:
index.html — this file
Icon/*.png — your Capture/Ability icon art
logic/cap.json, logic/cascade.json, ... — your raw export files, unmodified, named to match KINGDOM_MANIFEST
When this page is opened over http(s):// (which GitHub Pages does, but a double-clicked local file does not), it automatically fetches each file in /logic/ — so once it's pushed and Pages is turned on (repo Settings → Pages → Deploy from branch), all 17 kingdoms just show up with no manual upload step and no index file to keep in sync.
